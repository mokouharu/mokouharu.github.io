mac中本地nginx启动和反向代理

Posted on 2017-07-07
  • nginx 的配置文件nginx.conf 路径是在：/usr/local/etc/nginx/
  • 启动nginx服务的路径是在：/usr/local/Cellar/nginx/1.10.1/bin/
    • 启动服务 ： sudo nginx

https 反向代理 本地启动的http服务

  • 先启动nginx : sudo nginx

    • 启动后在浏览器中访问 localhost:81 , 如果出现 welcome to nginx ! 就表示nginx 启动成功。 （端口号是从自己在nginx.conf中设置的端口号获取的）

  • 访问某个需要https反向代理的项目， 以母婴项目为例

  • 在项目中启动本地服务： npm run dev

  • 然后在浏览器中访问 某个配置好的页面地址： https://localhost/Mzhuanzhuan/***/index.html#/home-recommend?webview=zzn

    • 为啥域名是localhost ? —— 是因为在nginx.conf 中配置的https 服务是 localhost

    • 如果不想写域名为localhost，可以在host中代理 127.0.0.1 对应域名
